The command line. One of the most important parts of the Linux experience. It is extremely important that any Linux user (or users of any OS at all) learn how the commandline works.

Terminal Emulators

Terminal emulators are programs used to access the terminal on a regular Desktop. Without these programs, users would be locked to just their Desktop with no lower level system access. Check your distro and Desktop Enviornment to see what your terminal emulator is called.

If you are using a server distro, the commandline will be opened by default, you don't need to worry about a terminal emulator.

Basic Commands

cd stands for Change Directory. It allows you to change the folder you are executing commands in.

Example: cd ~ (~ stands for your home directory, so this commands moves your current location to the home directory)

ls stand for LiSt. This command allows you to list every file and directory inside the one you are currently in.

rm stands for ReMove. this command lets you remove a file or directory.

Example: rm file.txt (removes a file in the current directory named "file.txt")

Example: rm -rf folder (removes a directory named "folder")

Example: rm -rf / (NEVER RUN THIS!!! IT DELETES YOUR ENTIRE SYSTEM!!!)

cp stands for CoPy. Allows you to copy a file from one location to the next

Example: cp file.txt ~/.config (copies a file named "file.txt" from your current directory to the ".config" directory stored in home)

mv stands for MoVe. Similar to cp but moves the file instead of copying it

Example: mv file.txt ~/.config (moves a file named "file.txt" from your current directory to the ".config" directory stored in home)

Using root is essential for some commands, since they may require higher privelidges than your default user has. If you run into a command that requires root, make sure to read it since it may mess up your system!

sudo runs the following command as the root user, requiring you to enter your password first

Example: "sudo apt install firefox" (installs firefox as the root user if your system uses apt as its package manager)

su allows you to run multiple commands as the root user. Not much else to say

If you want to learn more commands or get into more detail, the Arch Linux Wiki is an amazing resource, even if you aren't using Arch Linux
