Distributions (or more commonly, distros) are different releases and derivations of Linux all maintained by different groups of people. Distros each contain the basic core libraries and commands of linux, but add their own layers on top of it in order to differentiate it. One distro could have an easy to grasp GUI installer alongside coming with utilities like a pdf reader or a browser, while another distro could give you to complete bare minimum and require you to install everything yourself.

Distros can also be based off of other distros. For example, the biggest distro Ubuntu is based off of Debian. These 2 distros are very different, but they share a lot of features

Distros usually come with a Desktop Enviornment. You can think of a Desktop Enviornment as the core featureset and look of a desktop. A majority of distros only give one option of Desktop Enviornment to install, but some of the more customizable distros can install with any Desktop Enviornment you wish.

Here are some of the major distros!

Debian is a distro based around pure stability. Debian is extremely slow to put out non-security updates, usually taking around 3-4 years per version. This is to allow for maximm stability and minimal breakage.

Upsides

  • Easy to use installer
  • Extremely Stable
  • APT (Debian's package manager) is very well supported
  • Amazing for servers if installed minimally

Downsides

  • Slow to support newer hardware (thanks to stability)
  • Updates (when they actually release) can cause breakage
  • Minimal support for more common window managers such as Hyprland

Ubuntu is a Debian based distro focused on user friendlyness and security. It comes with the GNOME Desktop enviornment.

Upsides

  • Easy to use installer
  • Stable
  • Comes with APT package manager
  • Most commonly used distro, supported by majority of hardware
  • Corporate Backed
  • Many variants for people who want a specific desktop or have a specific use case

Downsides

  • Comes with the Snap package manager, which is very unstable
  • People may not like how deep corporate interested is rooted in it

Mint is a Debian based distro entirely based on being as user friendly as possible. It comes with either the Cinnamon or XFCE Desktop Enviornment depending on user choice.

Upsides

  • Easy to use installer
  • Ubuntu AND Debian bases, user chooses which they want
  • Comes with both APT and its own GUI software manager
  • Good support for almost every system

Downsides

  • Limited customizability
  • More advanced users may not enjoy using it as much

Fedora is a simple to use, user friendly distro

Upsides

  • Easy to use installer
  • Many "spins" which add different features while keeping the core
  • Backed by the Red Hate corporation and IBM

Downsides

  • Limited customizability
  • More advanced users may not enjoy using it as much
  • Requires some setup with proprietary drivers

Arch is a rolling release distro, meaning it doesn't update its packages all at once, instead opting to individually update each package as soon as a new release is available. Arch is also extremely customizable, coming with only the bare minimum needed to install.

Upsides

  • Extremely customizable
  • Tons of community support through the Arch User Repository
  • Updates are extremely frequent

Downsides

  • Installer is entirely command line
  • Rolling Release causes some updates to cause system breakage
  • Very unfriendly to new users
  • Requires lots of research and debugging

CachyOS is an Arch based distro with its main selling point being the custom kernel. Its kernel sacrifices older hardware compatability for optimizations on newer x86 CPUs.

Upsides

  • Installer is easy and customizable
  • Tons of community support through the Arch User Repository
  • Updates are extremely frequent
  • Custom kernel provides performance boost on newer CPUs

Downsides

  • Rolling Release causes some updates to cause system breakage
  • Can require some commandline usage
  • Requires research and debugging sometimes
  • Custom kernel has significant issues with older hardware

Endeavor is an arch based distro focused on user friendlyness and ease of install.

Upsides

  • Installer is easy to use
  • Tons of community support through the Arch User Repository
  • Updates are extremely frequent

Downsides

  • Rolling Release causes some updates to cause system breakage
  • Can require some commandline usage
  • Requires research and debugging sometimes

SteamOS is an arch based distro made purely for Steam hardware. It comes as an Atomic distro, meaning potentially dangerous commands are locked away so users can't easily brick the system.

Upsides

  • Backed by Valve, one of the biggest gaming companies in the world
  • Lots of community support
  • Comes preinstalled on all Steam hardware
  • Supports the ARM architecture

Downsides

  • Doesn't work well outside of Steam hardware

Gentoo is a distro focused on pure customizability, with little care for user friendliness. It features a flag system, allowing you to enable and disable parts of certain packages and system composnents in order to fine tune your system to be exactly what you want.

Upsides

  • Flag system allows for creation of extremely lightweight and finetuned systems
  • Most customizable distro
  • Comes with both APT and its own GUI software manager

Downsides

  • Difficult install, requiring the user to do everything themselves
  • Compiles all packages from source, meaning installation and updates can take extemely long on older hardware

ChromeOS is a gentoo based distro made for Chromebook devices.

Upsides

  • Extreme ease of use
  • Good support for ARM CPUs

Downsides

  • Heavily integrated with Google services
  • Not customizable in any way
  • Heavily browser based
  • Locks away many basic Linux functions

NixOS is a distro built off the NIX package manager, with the main feature being system reproducability. Instead of a traditional package manager, NIX uses configuration files to rebuild your system the exact way its configured. You can copy these configuration files to any NixOS system and it will essentially replicate your system.

Upsides

  • Simple installer
  • Extremely configurable
  • Has the largest official package repository out of any distro
  • Easy to use rollback system to remove changes that may have broken the system
  • Creates a reproducable system
  • Able to use older versions of packages with a 1 line configuration change

Downsides

  • Requires learning the NIX programming language to use

LFS is not technically a distro, rather a guide on how to create Linux From Scratch. This creates a distro fine tuned to the exact needs of the user, at the cost of the user having to create everything themselves

Upsides

  • You get to do EVERYTHING yourself

Downsides

  • You have to do EVERYTHING yourself
